# 아파트 게시물 관리 시스템 계획

## 목적

주차단속AI와 분리된 독립 시스템으로 안내문, 공고문, 전입/전출 안내, 상업게시물의 게시 기간과 철거 이력을 관리한다. 나중에 주차단속AI와는 같은 포털 메뉴, 같은 아파트 코드, 같은 사용자 권한 체계로 연결할 수 있게 설계한다.

## 1차 구현 범위

- 게시물 등록, 조회, 수정, 삭제
- 분류: 안내, 공고문, 전입/전출, 상업게시물, 기타
- 위치: 동, 층, 게시판명 등 자유 입력
- 게시 시작일, 게시 종료일, 철거 예정일
- 게시 사진 첨부
- 철거 완료 처리와 철거 사진/메모 기록
- 자동 상태 계산: 게시 예정, 게시 중, 기간 만료, 철거 대상, 철거 완료, 무단 게시
- 오늘 확인할 철거 대상 중심의 모바일 카드 화면

## 운영 방식

1. 관리자가 게시물을 등록한다.
2. 시스템이 날짜에 따라 게시 상태를 자동 계산한다.
3. 현장 직원은 `철거 대상`, `기간 만료`, `무단 게시`를 우선 확인한다.
4. 철거 후 사진과 메모를 남기고 `철거 완료` 처리한다.
5. 필요 시 전체 목록에서 수정 또는 삭제한다.

## 나중에 주차단속AI와 연결하는 방법

- 주차단속AI 첫 화면에 `게시물 관리` 버튼 추가
- 게시물 관리 화면에 `주차단속AI` 버튼 추가
- 공통 필드 유지: `site_code`, `username`, `role`
- 장기적으로는 통합 포털에서 두 시스템을 메뉴로 묶는다.

## 확장 후보

- Excel/PDF 철거 대장 출력
- 게시 종료 1~3일 전 알림
- 위치별 게시판 QR 코드
- 무단 상업게시물 반복 업체 통계
- 승인 요청/승인 완료 워크플로우
